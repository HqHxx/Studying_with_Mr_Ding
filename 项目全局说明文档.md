## 一、项目概述与技术栈

### 1.1 项目定位

**知识学爆** 是一款基于大语言模型（LLM）的智能英语学习桌面应用。它通过 AI 重写英文文章并植入目标词汇，生成中英双语对照的学习材料（Markdown + PDF + EPUB），帮助用户在真实语境中记忆单词。

### 1.2 技术栈

| 层级 | 技术/框架 | 版本要求 | 用途 |
|------|----------|---------|------|
| **UI 框架** | `customtkinter` | ≥5.2.0 | 现代化深色主题桌面 GUI |
| **AI 调用** | `openai` (Python SDK) | ≥1.0.0 | 兼容 OpenAI 协议的 LLM API 调用（豆包、DeepSeek 等） |
| **PDF 生成** | `fpdf2` | ≥2.7.9 | 护眼暗黑主题 PDF 排版输出 |
| **EPUB 打包** | `ebooklib` + `markdown` | ≥0.18 / ≥3.5 | 电子书生成 |
| **数据存储** | `SQLite3` (内置) | — | 多级别词库与用户学习进度 |
| **打包工具** | `PyInstaller` | — | 构建 Windows 可执行文件 |
| **爬虫工具** | `requests` + `BeautifulSoup4` + `Selenium` | — | 语料库自动采集（可选） |

### 1.3 运行环境

- **操作系统**：Windows（主要目标），兼容 Linux/macOS（需调整字体路径）
- **Python 版本**：3.10+
- **网络要求**：需要稳定的 LLM API 访问（支持火山引擎、OpenAI、本地模型等）

## 二、核心业务逻辑

### 2.1 学习工作流总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户操作层                                │
│  选择难度(CET-4/6/考研/托福) + 题材(history/science) + 生成篇数   │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                      三级 AI 流水线 (ContentEngine)              │
├─────────────────────────────────────────────────────────────────┤
│  Step 1: 智能选词 (Fast Model)                                   │
│  ├─ 从候选词库随机抽取 100 个单词                                │
│  ├─ 发送文章片段 + 候选词给轻量级模型                             │
│  └─ 模型挑选 25~50 个最适合语境的单词                             │
├─────────────────────────────────────────────────────────────────┤
│  Step 2: 深度重写 (Core Model)                                   │
│  ├─ 从语料库选取未使用过的文章                                    │
│  ├─ 发送原文 + 目标词汇给强推理模型                               │
│  └─ 模型重写为 600~800 词高质量英文长文，无痕植入加粗词汇          │
├─────────────────────────────────────────────────────────────────┤
│  Step 3: 逐段精译 (Fast Model)                                   │
│  ├─ 发送英文长文给轻量级模型                                      │
│  └─ 模型输出中英双语对照：英文段落 → 空行 → 中文翻译 → 空行...    │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                      输出与归档层                                │
│  ├─ Markdown 原文 → archives/markdown/                           │
│  ├─ 暗黑 PDF    → archives/pdf/                                  │
│  └─ EPUB 电子书 → archives/epub/ (批量打包)                      │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                      进度更新层                                  │
│  ├─ 数据库标记已学单词 (status = 1)                              │
│  ├─ 数据库标记已使用主题                                         │
│  └─ 本地 JSON 记录已使用文章标题 (used_articles.json)             │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 单词展示逻辑

- **候选词池**：从 SQLite 数据库中按级别（CET-4/6/考研/托福）随机抽取 `status=0`（未学）的单词，上限 300 个
- **智能筛选**：Fast Model 根据文章前 300 字符的语境，从候选词中挑选 25~50 个最相关的单词
- **植入展示**：Core Model 重写时将这些单词用 `**加粗**` 形式自然融入文章，用户在阅读时通过加粗识别生词

### 2.3 "记忆算法"说明

本项目**没有采用艾宾浩斯或 SM-2 等复杂记忆算法**，而是采用简单的"语境 exposure + 手动重置"策略：

1. **首次 exposure**：单词被选中并植入文章后，自动标记为 `status=1`（已学）
2. **避免重复**：已学单词不会再被抽入候选词池
3. **进度追踪**：右上角实时显示当前级别的"已掌握 / 待学习"数量
4. **手动重置**：用户可随时重置某级别或全部学习进度，让单词重新进入候选池

> 设计哲学：依赖 AI 生成的丰富语境实现"自然记忆"，而非机械复习。如需间隔重复，可后续集成 Anki 导出功能。

### 2.4 用户学习进度更新机制

| 操作 | 数据库变化 | 文件变化 |
|------|-----------|---------|
| 生成文章成功 | `words.status = 1`（已学） | — |
| 生成文章成功 | `used_topics` 表新增记录 | `used_articles.json` 追加标题 |
| 重置单词进度 | `words.status = 0`（按级别或全部） | — |
| 重置文章记录 | `used_topics` 表清空 | `used_articles.json` 清空为 `[]` |

## 三、数据流与存储

### 3.1 词库数据来源

```
原始词库文本文件 (Tab 分隔)
    ├── cet4_words.txt          (CET-4: 4,537 词)

            │
            ▼

            │
            ▼
    SQLite 模板数据库: data/cet4_words.db
    (表结构: word | phonetic | definition | level | status)
```

### 3.2 语料库数据来源

| 来源 | 数量 | 获取方式 |
|------|------|---------|
| `local_corpus.json` 内置 | 662 篇 | 项目自带（history: 248, science: 414） |
| Smithsonian Magazine | 可扩展 | `tools/fetch_smithsonian.py` (Selenium 爬虫) |

| World History Encyclopedia | 可扩展 | `tools/auto_fetch.py` (requests + BS4) |

### 3.3 用户学习记录存储

#### 3.3.1 SQLite 数据库 (`data/学习进度词库.db`)

运行时自动从模板复制，包含两张表：

**`words` 表** — 词库与进度
```sql
CREATE TABLE words (
    word TEXT,
    phonetic TEXT,
    definition TEXT,
    level TEXT NOT NULL DEFAULT 'CET-4',  -- CET-4 / CET-6 / 考研 / 托福
    status INTEGER NOT NULL DEFAULT 0,     -- 0=未学, 1=已学
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (word, level)
);
```

**`used_topics` 表** — 已使用文章主题
```sql
CREATE TABLE used_topics (
    topic TEXT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.3.2 JSON 文件

| 文件 | 用途 | 格式 |
|------|------|------|
| `.api_config.json` | API 配置持久化 | `{"profiles": {...}, "last_used": "..."}` |
| `used_articles.json` | 已使用文章标题（冗余备份） | `["title1", "title2", ...]` |
| `local_corpus.json` | 本地语料库 | `[{"title": "...", "content": "...", "category": "..."}]` |

## 五、关键设计决策

### 5.1 为什么用双模型策略？

| 步骤 | 模型类型 | 原因 |
|------|---------|------|
| 选词 + 翻译 | Fast Model（轻量/便宜） | 任务简单，不需要强推理，节省 Token |
| 文章重写 | Core Model（强推理） | 需要地道表达、复杂句式、无痕植入，必须高质量 |

### 5.2 为什么用 SQLite 而非云数据库？

- **隐私优先**：用户学习数据完全本地存储，不上传任何服务器
- **零配置**：无需安装额外数据库服务，开箱即用
- **单用户场景**：桌面应用天然适合嵌入式数据库

### 5.3 为什么语料库需要本地缓存？

- **降低 API 成本**：文章主题由本地随机选择，仅重写内容调用 LLM
- **避免重复**：`used_topics` 机制确保同一篇文章不会被重复改写
- **离线可用**：选文阶段无需联网，只有生成阶段需要 API

## 六、扩展与二次开发建议

### 6.1 常见扩展方向

| 方向 | 涉及文件 | 难度 |
|------|---------|------|
| 新增词库级别 | `tools/init_db.py`、`main.py` 下拉框 | ⭐ 低 |
| 新增语料来源 | `tools/` 新增爬虫脚本 | ⭐⭐ 中 |
| 集成 Anki 导出 | 新增 `anki_generator.py` | ⭐⭐ 中 |
| 间隔重复算法 | `db_manager.py` 新增复习字段 | ⭐⭐⭐ 高 |
| 多语言支持 | `content_engine.py` Prompt 改造 | ⭐⭐⭐ 高 |

### 6.2 调试技巧

- **查看 AI 原始输出**：`content_engine.py` 中设置 `is_debug=True`，终端会打印完整 Prompt 和流式输出
- **检查数据库状态**：直接用 SQLite 浏览器打开 `data/学习进度词库.db`
- **测试 API 连通性**：在 UI 中填写配置后点击保存，观察日志是否有报错
